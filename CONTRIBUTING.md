# Contribution Guidelines

Thank you for considering a contribution to this list! Please take a moment to review these guidelines before submitting.

## Adding an Entry

1. **Fork** the repository and create a new branch for your addition.
2. Add your entry to the most relevant section of `README.md`.
3. Use the following format for each entry:

   ```
   - [Name](https://link.to/resource) - A short, clear description of what makes it awesome.
   ```

4. Keep descriptions concise — one sentence is ideal.
5. Make sure the link works and points to the canonical or most useful page for the resource.
6. Place your entry in **alphabetical order** within its section.
7. Submit a **pull request** with a meaningful title and description.

## Quality Standards

- Only suggest items you genuinely find awesome and would recommend to others.
- Prefer actively maintained projects and up-to-date resources.
- Avoid duplicate entries — check the list before submitting.
- Do not add items that are promotional or primarily self-serving.

## Suggesting a New Section

If no existing section fits your entry, feel free to propose a new one in your pull request. Include at least **two** entries for the new section.

## Reporting Issues

If you find a broken link or outdated information, please open an issue or submit a pull request with a fix.
